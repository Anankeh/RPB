DR 2
Ransomware on File Server
Datto BCDR Unitrends
Initial Incident
  • An employee downloads a malicious attachment from a phishing email
  • Payload executes silently and begins encrypting files on a shared network drive
  • Because the file server is widely accessed, ransomware spreads across all mapped drives but does NOT affect the OS volume
  • Within minutes, the file server and connected storage volumes are fully encrypted
Immediate Impact
  • All shared folders become inaccessible or display encrypted file extensions
  • Employees across departments lose access to critical documents (finance, HR, operations)
  • Line-of-business applications relying on file shares begin to fail
  • A ransom note appears demanding payment for decryption keys
  • Productivity halts as users cannot open or save files
Question

Although it is highly unlikely they would have both BCDR and Unitrends deployed for this environment — what would our restore solutions be for each product?

Address both Datto BCDR and Unitrends separately in your answer.

DR 3
Salesforce Data Corruption
SaaS Protection / Spanning
Initial Incident
  • The company integrates Salesforce with a new third-party marketing automation tool
  • During initial setup, a misconfigured sync rule is deployed
  • The integration begins bulk updating records incorrectly:
    • Overwriting account details
    • Setting contact fields to blank or incorrect values
    • Updating opportunity stages improperly
  • Thousands of records are modified within minutes before the issue is detected
Immediate Impact
  • Sales pipeline becomes unreliable due to incorrect opportunity data
  • Account and contact records lose critical information
  • Reporting and forecasting dashboards become inaccurate
  • Sales and customer success teams cannot trust CRM data
  • Business operations slow down due to uncertainty and manual verification
Question

Which solution should they have had employed to protect against this scenario, and what kind of restoration would they be looking to perform?

DR 4
Lost Laptop with Un-synced Files
Endpoint Backup
Initial Incident
  • A sales executive is traveling and working offline (on a plane and later in a hotel)
  • They create and modify several important files:
    • Client proposals
    • Pricing spreadsheets
    • Contract drafts
  • Files are saved locally on Desktop and Documents — not yet synced to Microsoft 365 or cloud storage
  • Later that day, the laptop is lost or stolen (left in a taxi or airport)
Immediate Impact
  • Device is no longer accessible
  • Critical business files exist only on the lost laptop
  • Work completed during travel appears to be permanently lost
  • Potential delays in sales cycles and client deliverables
  • Security concern due to sensitive data on a missing device
Question

Which solution(s) would help protect against this scenario? What kind of restore would you perform to get the user's device and data back up and running?

DR 5
Large Phishing Attack
Datto BCDR SaaS Protection Endpoint Backup
Initial Incident
  • A user receives a convincing phishing email and enters credentials into a fake login page
  • The attacker gains access to:
    • Network credentials (servers and file shares)
    • Microsoft 365 account (email, OneDrive, SharePoint)
  • The attacker then:
    • Launches ransomware across file servers and virtual machines
    • Deletes and modifies files in SharePoint and OneDrive
    • Corrupts locally stored files on both in-office and remote endpoints
Immediate Impact
  • Servers: File servers and applications are encrypted; business-critical systems go offline
  • Microsoft 365: Files deleted/overwritten in SharePoint/OneDrive; email account compromised
  • User Devices: Local files encrypted or corrupted for both in-office and work-from-home users; non-synced work is lost
  • Business: Organization-wide downtime; high risk of permanent data loss across all environments
Question

The partner already has Datto BCDR deployed for office users and Endpoint Backup for remote workers. What other backup solutions could they have employed? What restore types would you deploy for BCDR and each additional product to fully restore the environment?

DR 6
Critical Application Server Failure
Unitrends
Initial Incident
  • During a routine patch deployment, a critical VMware application server crashes unexpectedly
  • The server hosts the firm's client reporting and transaction processing applications
  • Attempts to reboot the server fail — OS and application data are corrupted
  • IT discovers the virtual disk is damaged and may require a full rebuild
Immediate Impact
  • Employees cannot access client reporting applications
  • Daily financial transactions cannot be processed
  • Customer-facing dashboards go offline
  • Revenue-sensitive operations are at risk if downtime extends
  • Manual workarounds are not feasible due to data dependencies
Question

The company uses Unitrends as its backup solution. What steps would you take to resolve this DR situation? What could they have done in advance to increase their RTO performance?

DR 7
Hypervisor Failure & Accidental SaaS Deletion
Kaseya Unitrends SaaS Protection / Spanning (M365)
Initial Incident
  • 1. Hypervisor Failure (On-Prem): During routine maintenance, a VMware ESXi host crashes due to a corrupted firmware update. The host contains:
    • Case management applications
    • Client document servers
    • Internal collaboration platforms
  • 2. Accidental SaaS Deletion (M365): Simultaneously, an admin cleaning up old SharePoint sites accidentally deletes an active project folder and multiple OneDrive accounts
Immediate Impact
  • On-Prem: Applications and file servers go offline; employees cannot access client documents; case tracking is disrupted
  • Microsoft 365: Users lose access to SharePoint project files and OneDrive content; collaboration halted; native retention policies cannot fully recover deleted items
Question

What would a disaster recovery plan look like for this scenario? Address both the Unitrends restore strategy for the on-prem environment and the SaaS Protection restore strategy for Microsoft 365.

DR 8
Non-Critical Restores for BCDR
Datto BCDR
Initial Incident
  • An IT administrator runs a cleanup script on the company's shared file server
  • A misconfigured command deletes entire project directories across multiple mapped drives
  • These directories contain:
    • Client deliverables
    • Financial documents
    • Internal strategy files
  • The deleted data spans an entire server volume — not just individual folders
Immediate Impact
  • Employees lose access to critical files across multiple departments
  • Sensitive folders with granular NTFS permissions (restricted by team, role, and client) are gone
  • Even if files were recovered manually, rebuilding permissions would be extremely time-consuming and error-prone
  • Risk of unauthorized access if permissions are restored incorrectly
  • Compliance issues for client data
  • Business operations slow significantly due to missing data and access control concerns
Question

What type of Datto BCDR restore would best be used in this situation — and why is that restore type specifically suited to preserve what was lost?
