RPO & RTO Fundamentals

RPO — Recovery Point Objective

Defines the maximum acceptable amount of data loss, measured in time (e.g., 4 hours of data), determining backup frequency.

Key question: "If disaster strikes right now, how much data can we afford to lose?" — that answer IS your RPO.

RTO — Recovery Time Objective

Defines the maximum tolerable downtime to restore systems after a failure.

Key question: "How long can the business survive without this system operational?" — that answer IS your RTO.

How RPO and RTO drive technology decisions

RPO drives backup frequency

A 1-hour RPO requires backups every 60 minutes or less. A 24-hour RPO allows nightly backups.

RTO drives recovery architecture

A 15-minute RTO requires instant virtualization. A 4-hour RTO may allow a full bare-metal restore.

Datto BCDR — Business Continuity Mindset

  • Uptime First — Prioritize operational continuity over backup storage
  • Phased Recovery — Virtualize immediately, restore permanently afterward
  • Multiple Recovery Paths — Local, Cloud, Hypervisor, Hybrid
  • RPO / RTO Alignment — Recovery decisions driven by business impact
  • Operational Flexibility — Choose recovery based on incident scope
  • Validated Recoverability — Test backups before incidents occur
  • Predictable Outcomes — Maintain consistent recovery performance during disruptions

Datto BCDR provides four distinct virtualization topologies, allowing administrators to dynamically align their recovery strategy with the scope and severity of the outage:

1. Local Virtualization
  • Environment: Local on-site appliance compute.
  • Advantage: Absolute fastest speeds; zero internet reliance.
  • Use Case: Local hardware failure / BSOD.
2. Cloud Virtualization
  • Environment: Hosted in secure Datto Cloud.
  • Advantage: Remote access if office is destroyed.
  • Use Case: Fire, flood, total site disaster.
3. Hypervisor Virtualization
  • Environment: Backups virtualized onto existing VMware vSphere or Hyper-V clusters.
  • Advantage: Leverages client's dedicated enterprise compute, RAM, and storage IOPS.
  • Use Case: Enterprises minimizing compute load on the Datto appliance.
4. Hybrid Virtualization
  • Environment: VM executes in cloud/hypervisor, routed via secure VPN tunnel.
  • Advantage: "Local-like" experience for users. Continuous backup of the failover VM is supported.
  • Use Case: Outages requiring transparent continuity and AD communication.

Non-Critical Restore Operations

File Restores

Restore individual files or folders from a selected recovery point. Available via local speed or Cloud redundancy.

Volume Restores via iSCSI

Created locally on the Datto appliance; accessed over the local network via iSCSI target. Preserves original NTFS permissions (ACLs) when using Robocopy for migration.

Granular Restores

Integration: Kroll Ontrack. Drill into snapshots for targeted, item-level recovery for Exchange, SQL, or SharePoint.

Typical Disaster Recovery Restores

Image Export

Export a recovery point to:

  • vSphere: VMDK format
  • Hyper-V: VHD or VHDX format
  • Delivered via USB drive or network share

Use when migrating to a new hypervisor environment or when physical hardware is replaced.

Bare Metal Restore (BMR)

Re-image a protected system completely from a snapshot using Datto Utilities.

  • Restores OS, applications, and data to bare hardware
  • Can target identical or dissimilar hardware
  • Use when the OS cannot be repaired in place

Best for physical server replacement after hardware failure or ransomware that destroyed the OS.

Other Disaster Recovery Restores

Rapid Rollback: Targeted System Restoration

Rapid Rollback provides a highly efficient mechanism to revert a system to a known-good state without initiating a full bare-metal rebuild. Instead of rehydrating the entire disk image, the system operates by replacing only the changed portions — not rebuilding the entire system.

This block-level delta restoration is significantly faster than traditional restores, particularly for ransomware, bad updates, or system corruption. Available via the Datto Utilities environment.

Meeting RPO and RTO Objectives
RPO (Data Loss Minimization)

Leverages Inverse Chain Technology™ — each backup snapshot serves as a fully constructed, bootable recovery point. Data loss is mathematically capped at the backup interval.

RTO (Downtime Minimization)

Immediate uptime via Instant Virtualization can reduce RTO from hours to minutes. Transition back to production via Image Export or BMR.

What is a Replica?

The VM replica feature provides a quick way to recover a failed server. It creates a virtual machine replica of the original VM on a secondary hypervisor, then keeps this replica continuously up-to-date by applying incremental backup jobs as they run.

In the event of a disaster, you can bring this replica online immediately to assume the role of the failed VM — temporarily or as a permanent replacement.

Replica is always a functional VM — just powered off
Minimal RTO — no full restore required on failover
Can become a permanent replacement for a failed server
Kept current via incremental backup synchronization

SaaS Protection & Spanning

SaaS Backup: Comprehensive Data Recovery

Securely backs up data stored in cloud environments with regular automated backups and granular restore capabilities.

Platforms Covered
  • Microsoft 365: Email, Calendar, Contacts, OneDrive, SharePoint, Teams
  • Google Workspace: Gmail, Drive, Calendar, Contacts, Shared Drives
Key Benefit

Multiple automated backups per day with granular restores — from a single deleted email to an entire user account. Covers accidental deletion, insider threats, ransomware, and platform outages.

Understanding SaaS Backup Needs

Accidental Deletion

Users delete critical emails, files, or entire accounts. Cloud providers have limited and time-capped recovery windows.

Insider Threats

Malicious or departing employees exfiltrate or destroy data. Provider logs the event — but does not restore it.

Ransomware

Encrypts files synced to OneDrive/Drive. Propagates across all connected devices before detection.

Shared Responsibility Model
Cloud Provider Guarantees:

Infrastructure availability, uptime, and physical security. M365 and Google Workspace SLAs cover the platform — NOT your data.

Your Responsibility:

Data backup, restoration, and retention. If you delete a file or get hit by ransomware, you are responsible for recovery.

Key Features of SaaS Backup / Spanning

Automated Daily Backups

Regular, automated snapshots of email, calendar, contacts, OneDrive, SharePoint, Teams, Google Drive, and Gmail — multiple times per day.

Flexible Data Retention

Customizable retention periods to meet regulatory compliance requirements — HIPAA, GDPR, SOC 2, and more.

Granular Restore Capabilities

Single items or full user accounts restored rapidly. Recover one email, one file, or the entire account — without restoring everything.

Search & Export

Locate and recover lost data or export for compliance and legal hold purposes. Search across backup snapshots by keyword, date, or sender.

Performing Data Restores

1
Identify the Snapshot

Select the backup snapshot taken before the data loss occurred. Timeline view makes it easy to find the right recovery point.

2
Browse or Search

Quickly browse or search backups for specific files, folders, or emails. Filter by keyword, date range, sender, or file type.

3
Initiate Restore

Restore directly to the original location or to an alternative user account. No disruption to other users or active data.

Real-World Scenario

Ransomware Recovery

1
Identify Last Clean Backup

Use ransomware detection to locate the last backup point before encryption began. This is your recovery baseline.

2
Select Affected Users and Files

Identify which users and files were encrypted. Select only the affected data for targeted restoration — minimizing restore scope.

3
Restore Data Seamlessly

Restore data to minimize downtime and operational disruption. Users regain access without manual file rebuilding.

Key Insight: The powerful combination of detection and automated response is crucial for effective cybersecurity incident management. Backup is not just recovery — it is the last line of defense.

NIST CSF — Recover Function
